── Method Details ──────────────────────────────────────────────

__________
PUB null()

 This is not a top-level object - placeholder to indicate driver must be instantiated as OBJ.

______________________
PUB init() : workerCog

 Initialize the unified filesystem driver (singleton).
 Allocates hardware lock and starts the worker cog.
 Pin assignments are fixed CON constants for the P2 Edge Module.
 Does NOT mount any filesystem -- call mount() after init().
 @returns workerCog - Cog ID (0-7) on success, -1 on failure

__________
PUB stop()

 Stop worker cog - cleanly shut down the worker cog and release resources.
 Safe to call even if not started.

_______________________
PUB mount(dev) : status

 Mount one or both filesystems (multi-cog safe).
 Requires init() to have been called first (worker cog must be running).
 Use DEV_SD to mount SD card, DEV_FLASH to mount Flash chip, DEV_BOTH for both.
 @param dev - Target device (DEV_SD, DEV_FLASH, or DEV_BOTH)
 @returns status - SUCCESS or negative error code

_________________________
PUB unmount(dev) : status

 Unmount one or both filesystems (multi-cog safe).
 Flushes pending data and updates filesystem metadata.
 @param dev - Target device (DEV_SD, DEV_FLASH, or DEV_BOTH)
 @returns status - SUCCESS or negative error code

_____________________________
PUB mounted(dev) : is_mounted

 Check if a device is currently mounted.
 @param dev - Target device (DEV_SD or DEV_FLASH)
 @returns is_mounted - TRUE if device is mounted, FALSE otherwise

________________________________
PUB canMount(dev) : is_mountable

 Non-destructive check if a device can be mounted (requires init, not mount).
 For SD: checks card presence and FAT32 validity.
 For Flash: scans blocks non-destructively (Phase 2+).
 @param dev - Target device (DEV_SD or DEV_FLASH)
 @returns is_mountable - TRUE if device appears mountable, FALSE otherwise

__________________________________
PUB cardWarnings() : warning_flags

 Get card capability and advisory flags discovered during mount.
 Returns bitmask of conditions probed at init time.
 Check after mount() to learn what the card actually supports in SPI mode.
   CW_CMD13_UNRELIABLE ($01) - CMD13 status checks disabled (broken SPI implementation)
   CW_CMD23_SUPPORTED  ($02) - CMD23 SET_BLOCK_COUNT confirmed working in SPI mode
 @returns warning_flags - Bitmask of CW_* constants (CW_NONE if no flags set)

______________________
PUB version(dev) : ver

 Get driver version for a device as encoded integer.
 Format: major * 10000 + minor * 100 + patch (e.g., 1_03_00 = v1.3.0).
 @param dev - Target device (DEV_SD, DEV_FLASH, or DEV_BOTH)
 @returns ver - Encoded version integer

___________________________
PUB versionStr(dev) : p_str

 Get driver version as a formatted "X.Y.Z" string pointer.
 @param dev - Target device (DEV_SD, DEV_FLASH, or DEV_BOTH)
 @returns p_str - Pointer to null-terminated version string (e.g., "1.3.0")

________________________
PUB format(dev) : status

 Format one device (destructive!).
 SD: full FAT32 format. Flash: cancel all active blocks.
 @param dev - Target device (DEV_SD or DEV_FLASH)
 @returns status - SUCCESS or negative error code

___________________
PUB sync() : status

 Flush buffers (multi-cog safe) - flushes pending writes without closing file.
 Use to checkpoint data during long operations.
 @returns status - SUCCESS or negative error code

 Create new file for writing (V3 multi-file) - creates and opens for writing.
 SD: returns error if file exists. Flash: opens for write (overwrites if exists).
 Flash: CWD prefix is automatically prepended to the filename.
 Use closeFileHandle() when done to flush data and finalize size.

 @param p_path - Pointer to null-terminated path string
 @returns handle - File handle (0 to MAX_OPEN_FILES-1) on success, negative error code on failure

____________________________________
PUB closeFileHandle(handle) : status

 Close file handle (V3 multi-file) - closes handle, flushing pending writes.
 For write handles, updates directory with final file size.
 After close, handle is invalid and slot is available for reuse.
 @param handle - Valid file handle
 @returns status - SUCCESS or negative error code

____________________________________________________
PUB readHandle(handle, p_buffer, count) : bytes_read

 Read from file handle (V3 multi-file) - reads up to count bytes into buffer.
 Returns actual bytes read (may be less at EOF). Returns 0 at EOF.
 Position advances by the number of bytes read.
 @param handle - Valid file handle open for read
 @param p_buffer - Pointer to destination buffer
 @param count - Maximum bytes to read
 @returns bytes_read - Bytes actually read (0 at EOF), negative on error

________________________________________________________
PUB writeHandle(handle, p_buffer, count) : bytes_written

 Write to file handle (V3 multi-file) - writes count bytes from buffer.
 File grows as needed with automatic cluster allocation.
 Position advances by the number of bytes written.
 @param handle - Valid file handle open for write
 @param p_buffer - Pointer to source data
 @param count - Bytes to write
 @returns bytes_written - Bytes actually written, negative on error

_____________________________________________________

 @param p_buffer - Pointer to destination buffer
 @param count - Bytes to read
 @returns status - PENDING, E_ASYNC_BUSY, or E_NOT_MOUNTED

______________________________________________________
PUB startWriteHandle(handle, p_buffer, count) : status

 Begin an asynchronous write. Returns immediately with PENDING (1).

 @param handle - Valid file handle open for write
 @param p_buffer - Pointer to source data
 @param count - Bytes to write
 @returns status - PENDING, E_ASYNC_BUSY, or E_NOT_MOUNTED

_______________________
PUB isComplete() : done

 Non-blocking poll for async operation completion. Safe to call repeatedly.
 @returns done - TRUE if async operation has completed, FALSE otherwise

________________________
PUB getResult() : status

 Get the result of an async operation and release the lock. If the operation
 hasn't finished yet, blocks until it does.
 CRITICAL: Drains stale COGATN before releasing lock to prevent silent
 data corruption in subsequent blocking calls.
 @returns status - Bytes read/written on success, or negative error code

__________________________
PUB cancelAsync() : status

 Wait for async operation to finish, discard result, release lock.
 Use when the caller no longer needs the result.
 @returns status - SUCCESS, or E_NO_ASYNC_OP

_________________________________________
PUB seekHandle(handle, position) : status

 Seek to position in file (V3 multi-file) - seeks to absolute byte position.
 Position must be <= file size (cannot seek past EOF).
 @param handle - Valid file handle
 @param position - Absolute byte position to seek to
 @returns status - SUCCESS or negative error code

_________________________________________________

PUB rename(dev, old_name, new_name) : status

 Rename file or directory (multi-cog safe).
 Flash: CWD prefix is automatically prepended to both names. Renaming a file
 to a different path prefix effectively moves it between directories. If this
 was the last file under the old prefix, that directory vanishes (Flash directories
 are not stored on disk -- they exist only as filename path prefixes).
 @param dev - Target device (DEV_SD or DEV_FLASH)
 @param old_name - Pointer to current name string
 @param new_name - Pointer to new name string
 @returns status - SUCCESS or negative error code

______________________________________
PUB deleteFile(dev, name_ptr) : status

 Delete file (multi-cog safe) - deletes file from current directory.
 Flash: CWD prefix is automatically prepended to the filename.
 NOTE: Flash directories are not stored on disk -- they exist only because files
 with matching path prefixes exist. Deleting the last file in a Flash directory
 causes that directory to vanish. This is an inherent property of the path-prefix
 emulation model, not an explicit step performed by this method.
 @param dev - Target device (DEV_SD or DEV_FLASH)
 @param name_ptr - Pointer to null-terminated filename string
 @returns status - SUCCESS or negative error code

PUB compactFile(dev, p_path) : status

 Relocate a file's clusters into a contiguous chain on disk. SD-only.
 The file must not be open (no active handles). If already contiguous, returns SUCCESS (no-op).
 Uses copy-then-free strategy: file is always readable during compaction.
 Read-back verification is performed after every copy.
 @param dev - Target device (must be DEV_SD)
 @param p_path - Pointer to null-terminated filename string
 @returns status - SUCCESS, E_FILE_NOT_FOUND, E_FILE_OPEN_FOR_COMPACT, E_NO_CONTIGUOUS_SPACE,
 E_VERIFY_FAILED, E_IO_ERROR

PUB isHighSpeedActive() : active

 Check if high-speed mode is active (running above standard 25 MHz).
 Uses 30 MHz threshold to account for P2 SPI clock divider rounding
 (e.g., 270 MHz P2 clock yields 33.75 MHz max SPI, not 50 MHz).
 @returns active - TRUE if running above standard speed
 - Reduced overhead from CS cycling and command processing
 CRITICAL: Multi-block write uses different tokens than single-block!
 - Each data block uses $FC (not $FE) as start token
 - Transmission ends with $FD stop token (not CMD12!)
 Protocol: CMD25 -> ($FC + 512 bytes + CRC + wait busy) x N -> $FD -> wait busy
